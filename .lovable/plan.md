## Standards Applied Per Challenge

Each challenge row follows `docs/NEW_CHALLENGE_STANDARDS.md` exactly:
- `is_active = false` on all 13 inserts (activation after stamps/audio verified)
- `price_cents`, `stripe_price_id`, `stripe_product_id` all null
- `total_miles` = mile of milestone 6 for each

Each milestone row:
- `order_index` 1–6 sequential
- `miles_required` matching the document's mile markers (milestone 1 always = 1 mile)
- `title` = person's name
- `stamp_title` = short unique title (derived from milestone name in doc, e.g. "Delta Born", "Washboard to Vision") — verified to not conflict with existing Women's History stamps
- `stamp_copy` = 1 concise sentence distilled from the multi-sentence block in the doc
- `historical_event` = the full 3-sentence block from the doc (this is the ElevenLabs narration text)
- `location_name` = specific geographic place derived from context of each milestone
- `latitude` / `longitude` = real coordinates for each location
- `stamp_mileage_display` = e.g. "1 MILE", "8 MILES"
- `audio_url` = null (auto-generated by trigger)
- `stamp_image_url` = null (generated via admin UI after insert)

### 1. `src/pages/ChallengeRoute.tsx` — add First Black Pioneers color theme

```text
case "pioneers":
  amber/bronze tones — distinct from gold, not rainbow
```

```text
if edition includes "First Black Pioneers" → return "pioneers"
```

Also update `getPricingEditionColor()` to map pioneers → "gold" (closest existing pricing color).

## Database Migration Strategy

Since this is a large multi-row insert (13 challenges + 78 milestones), it will be done as a single migration SQL file using two passes:
1. INSERT all 13 challenge rows (get back their generated UUIDs via `RETURNING id`)
2. INSERT all 78 milestone rows referencing each challenge's UUID

The migration uses `gen_random_uuid()` via PostgreSQL's `uuid_generate_v4()` function and assigns milestone UUIDs inline using explicit `uuid_generate_v4()` calls so the referencing is self-contained in one script.

`audio_url = null` on all milestones — the `trigger_milestone_audio_generation` database trigger fires automatically on each INSERT and calls the ElevenLabs edge function for each of the 78 milestones.
